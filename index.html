    Introduction

    Welcome to the Bmatic API! You can use our API to integrate your application with us. Now, trought our endpoints you can access to services of BMATIC’s branches from any application.

    You can download our specification and generate your own client in your preference language trought the code generator of swagger and speed up your integration with us.

    Improve your customers’ experience!

    Schemes: http, https

    Host: api.bmatic.com

    Base path: /v1

    Branches API

    Before you create tickets or bookings, you need a branch and sector provided by the endpoints of this section (The booking creation does not require a sector). You should consider using the recommended parameters for better performance.

    List branches

    Sample request:

    GET /v1/branches/branch?offset=9&limit=1 HTTP/1.1

    Sample response:

    HTTP/1.1 200 OK
X-Pagination-Count: 32
X-Pagination-Page: 10
X-Pagination-Limit: 1
Content-Type: application/json

[

        "id": "27b6",
        "name": "Hiper Central",

        "address": "Calle Beta 181 - 195, Callao",
        "latitude": -12.049919,
        "longitude": -77.0845193,

    Returns all branches with their sectors that fulfill the filters.

    Caution: You should consider the next:
    • The branches to return should to be marked with the flag exposed by the API from Bmatic configuration panel.
    • An empty collection is returned when no branch fulfill the filters.
    • Filter branches by geolocalization it is necessary to specify minLatitude, maxLatitude, minLongitude and maxLongitude. If any of these parameters is not included then all will be ignored.
    • To order the branches by proximity to the client location it is necessary specify latitude and longitude. If any of these parameters is not included then all will be ignored.
    Order: Branches are sorted by the shorter distance to client location and their short name.
    Remember: Customers prefer to attend the nearest branches, for that it is recommendable ever use longitude, latitude, minLatitude, maxLatitude, minLongitude and maxLongitude parameters.

    Endpoint

    GET /v1/branches/branch

    Query Params

    • latitude Float recomended
      Latitude component of customer location.

    • longitude Float recomended
      Longitude component of customer location.

    • minLatitude Float recomended
      Minimum latitude of the geographic area that has to contain the branches.

    • maxLatitude Float recomended
      Maximum latitude of the geographic area that has to contain the branches.

    • minLongitude Float recomended
      Minimum longitude of the geographic area that has to contain the branches.

    • maxLongitude Float recomended
      Maximum longitude of the geographic area that has to contain the branches.

    • fields List<String> recomended
      Entity fields that will included in the response(See available fields in the branch object definition). For example: fields=id,name,shortName.

    • offset Integer
      Position in pagination.

      Default value: 0
      Maximum value: 9999

    • limit Integer
      Number of items to retrieve.

      Default value: 10
      Maximum value: 50

    Responses

    • 200 Ok List<Branch>
      A list of branches. Pagination headers are included:

      • X-Pagination-Count: Total number of items.
      • X-Pagination-Page: Number of current page.
      • X-Pagination-Limit: Limit of items per page.

    • 400 Bad Request ValidationError
      One or more parameters are not valid, returns a description of validation failed.

    • 500 Internal Server Error Error
      An unexpected error has occurred, returns a simple error message.

    Create a new customer ticket and returns it.

    Caution: You should consider the following before create a ticket:
    • The channel used should support the requested service.
    • The requested branch should contain the requested sector.
    • The requested sector should support the requested service.
    • The customer should has no other ticket.
    • The customer category referenced by the categoryId should has a queue configured for the requested service and the channel used.

    Endpoint

    POST /v1/tickets/customer/{customerId}/tickets

    Path Params

    • customerId String
      Customer unique identifier.

    Request

    • channelId String required
      Unique identifier of the channel used. This value should be fixed in the client application.

    • branchId String required
      Unique identifier of the requested branch.

    • sectorId String required
      Unique identifier of the requested sector.

    • serviceId String required
      Unique identifier of the requested service.

    • docType String
      Type of document used to identify the client.

      Maximum length: 5

    • docNumber String
      Number of document used to identify the client.

      Maximum length: 20

    • phone String
      Customer phone for notifications associated to the ticket.

      Validation pattern: ^\+[1-9]{1}[0-9]{3,14}$

    Responses

    • 201 Created Ticket
      Successful creation, returns the new ticket.

    • 400 Bad Request ValidationError
      One or more parameters are not valid, returns a description of validation failed.

      An unexpected error has occurred, returns a simple error message.

    • The client referenced should has no other booking in state ACTIVE or PENDING for the same requested service.
    • The startTime and the endTime should to be available.

    This operation can throw the next validation errors:

    Code Field Message
    40001 channelId Channel not exists.
    40002 serviceId Service not exists.
    40003 branchId Branch not exists.
    40006 serviceId Service not supported by the channel.
    40007 serviceId Service not supported by the branch.
    40012 customerId Customer has another valid booking for the same service.
    40013 startTime Booking start time not valid.
    40014 endTime Booking end time not valid.
    40015 serviceId Service not available in the requested time.

    Model of branch object.

    Caution: The congestion, lowEstimatedTime and highEstimatedTime are calculated using the configuration of the associated branch type. You need configure the next:
    • Time range to consider to calculate the waiting time.
    • Desviation of waiting time.
    • Minimal waiting time of medium congestion.
    • Minimal waiting time of high congestion.
    If the branch is INACTIVE this fields are not returned.

    Fields

    • id String
      Branch unique identifier (Configured from Bmatic configuration panel).

    • name String
      Complete name of the branch.

      Maximum length: 50

    • shortName String
      Name to display in small devices.

      Maximum length: 20

    • address String
      Branch address.

      Maximum length: 100

    • latitude Float
      Latitude component of branch location.

    • longitude Float
      Longitude component of branch location.

    • status Enum
      Current status of the branch.

      Possible values:

      • ACTIVE: At least one sector is ACTIVE
      • INACTIVE: All sectors are INACTIVE

    • congestion String
      Flag of branch congestion.

      Possible values:

      • LOW: The average waiting time is lower to the minimal waiting time of medium congestion.
      • MEDIUM: The average waiting time is lower to the minimal waiting time of high congestion but equal or upper to the minimal waiting time of medium congestion.
      • HIGH: The average waiting time is equal or upper to the minimal waiting time of high congestion

    • lowEstimatedTime Integer
      Lower bound of the estimated waiting time. It’s equal to the average waiting time minus the configured desviation.

    • highEstimatedTime Integer
      Upper bound of the estimated waiting time. It’s equal to the average waiting time plus the configured desviation.

    • sectors List<Sector>
      List of the branch’s sectors. A correct configuration include at least one sector per branch.

      Unique items: true
      Minimum items: 1
