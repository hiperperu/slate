    Introduction

    Welcome to the Bmatic API! You can use our API to integrate your application with us. Now, trought our endpoints you can access to services of BMATIC’s branches from any application.

    You can download our specification and generate your own client in your preference language trought the code generator of swagger and speed up your integration with us.

    Improve your customers’ experience!

    Schemes: http, https

    Host: api.bmatic.com

    Base path: /v1

    Branches API

    Before you create tickets or bookings, you need a branch and sector provided by the endpoints of this section (The bookings creation not required a sector). You should consider use the recommended parameters for obtain a better performance in your application.

    List branches

    Sample request:

    GET /v1/branches/branch?offset=9&limit=1 HTTP/1.1

    Sample response:

    HTTP/1.1 200 OK
X-Pagination-Count: 32
X-Pagination-Page: 10
X-Pagination-Limit: 1
Content-Type: application/json

[
    {
        "id": "27b6",
        "name": "Hiper Central",
        "shortName": "Hiper",
        "address": "Calle Beta 181 - 195, Callao",
        "latitude": -12.049919,
        "longitude": -77.0845193,
        "status": "ACTIVE",
        "congestion": "MEDIUM",
        "lowEstimatedTime": 10,
        "highEstimatedTime": 14,
        "sectors": [
            {
                "id": "m8",
                "name": "Tower A",
                "shortName": "A",
                "status": "ACTIVE",
                "services": ["s9","s10","s30"]
            }
        ]
    }
]

    Returns all branches with their sectors that fulfill the filters.

    Warning: The branches to return should to be marked with the flag expose by API from the Bmatic configuration.
    Order: The branches are sorted by the better distance to client location and then by its shortName.
    Remember: The customers prefer to attend the nearest branches, for that it is recommendable ever use the parameters longitude, latitude, minLatitude, maxLatitude, minLongitude and maxLongitude.

    Endpoint

    GET /v1/branches/branch

    Query Params

    • latitude Float recomended
      Latitude component of customer location.
    • longitude Float recomended
      Longitude component of customer location.
    • minLatitude Float recomended
      Minimun latitude of the area that has to contain the branches.
    • maxLatitude Float recomended
      Maximum latitude of the area that has to contain the branches.
    • minLongitude Float recomended
      Minimun longitude of the area that has to contain the branches.
    • maxLongitude Float recomended
      Maximum longitude of the area that has to contain the branches.
    • serviceId String
      Unique identifier of service supported by the branch.
    • fields List<String> recomended
      Entity fields that will return at response. For example: fields=id,name,shortName.
    • offset Integer
      Position in pagination.

      Default value: 0
      Maximum value: 9999

    • limit Integer
      Number of items to retrieve.

      Default value: 10
      Maximum value: 50

    Responses

    • 200 Ok List<Branch>
      A list of branches. Pagination headers are included:
      • X-Pagination-Count: Total number of items.
      • X-Pagination-Page: Number of current page.
      • X-Pagination-Limit: Limit of items per page.
    • 400 Bad Request ValidationError
      One or more parameters are not valid, returns a description of validation failed.
    • 500 Internal Server Error Error
      An unexpected error has occurred, returns a simple error message.

    Create a new customer ticket and then returns it.

    Caution: You should consider the following before create a ticket:
    • The channel referenced by the channelId have to support the service requested.
    • The branch referenced by the branchId have to contain the sector referenced by the sectorId.
    • The sector referenced by the sectorId have to support the service requested.
    • The client referenced by the customerId should has no other ticket.
    • The customer category referenced by the categoryId should has a queue configured for the service requested and the channel used.

    Endpoint

    POST /v1/tickets/customer/{customerId}/tickets

    Path Params

    • customerId String
      Customer unique identifier.

    Request

    • channelId String required
      Channel unique identifier.
    • branchId String required
      Branch unique identifier.
    • sectorId String required
      Sector unique identifier.
    • serviceId String required
      Service unique identifier.
    • categoryId String required
      Customer category unique identifier.
    • name String
      Name or alias of client used for contact purpose.

      Maximum length: 50

    • document String
      Document number of customer.

      Maximum length: 20

    • phone String
      Customer phone for notifications.

      Validation pattern: ^+[1-9]{1}[0-9]{3,14}$

    • teller String
      Account name of teller associated to the customer. When this parameter is provided, the ticket generated should associate to the counter of the teller referenced.
    • messagesToCustomer List<String>
      Messages to print in ticket.
    • messagesToTeller List<String>
      Messages to show in counter.

    Responses

    • 201 Created Ticket
      Successful creation, returns the new ticket created.
    • 400 Bad Request ValidationError
      One or more parameters are not valid, returns a description of validation failed.
    • 500 Internal Server Error Error
      An unexpected error has occurred, returns a simple error message.
